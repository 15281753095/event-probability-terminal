# EPT API v1 Local Contract

Status: RC-6 OpenAPI-like local contract publication for the fixture-backed Phase 1 API.

This document publishes the local consumer contract for `GET /scanner/top` and
`GET /markets/:id/detail`. It is not a live vendor API specification, not a Polymarket upstream
schema, and not a trading API. It documents the local API gateway response surface used by the web
app and future local consumers.

Current contract version:

```text
ept-api-v1
```

## Base URL

Local development base URL:

```text
http://localhost:4000
```

Responses are JSON. The current contract is fixture-backed by default. Snapshot tests use a fixed
clock, so example timestamps from snapshots are deterministic; normal runtime `generatedAt` values
are generated by the API gateway.

## Shared Success Metadata

Successful scanner/detail responses include a `meta` object with these stable contract fields:

| Field | Type | Contract meaning |
| --- | --- | --- |
| `contractVersion` | `"ept-api-v1"` | Local response contract version. |
| `responseKind` | `"scanner_top"` or `"market_detail"` | Successful response kind. |
| `generatedAt` | string | API gateway generation timestamp. |
| `status` | `"ok"` | Successful response status. |
| `source` | `"polymarket"` | Current primary source boundary. |
| `mode` | `"fixture"` or `"live_public"` | Adapter source mode. The current default is fixture mode. |
| `isFixtureBacked` | boolean | Whether the response came from fixture-backed adapter data. |
| `isReadOnly` | `true` | The route does not place, cancel, settle, fund, or trade. |
| `isPlaceholderPricing` | `true` | Fair value, confidence, and edge are placeholder-only. |
| `message` | string | Human-readable read-only/placeholder explanation. |

Example:

```json
{
  "contractVersion": "ept-api-v1",
  "responseKind": "market_detail",
  "generatedAt": "2026-04-23T00:00:00.000Z",
  "status": "ok",
  "source": "polymarket",
  "mode": "fixture",
  "isFixtureBacked": true,
  "isReadOnly": true,
  "isPlaceholderPricing": true,
  "message": "Market detail is read-only and contract-backed. Pricing, confidence, and edge remain placeholders."
}
```

## Shared Typed Errors

Typed error responses use the shared `ApiErrorResponse` shape:

| Field | Type | Contract meaning |
| --- | --- | --- |
| `contractVersion` | `"ept-api-v1"` | Local response contract version. |
| `status` | `"not_found"`, `"unsupported"`, or `"fail_closed"` | Error status taxonomy. |
| `error` | `"market_not_found"`, `"unsupported_market"`, or `"out_of_scope"` | Error code taxonomy. |
| `message` | string | Human-readable explanation. |
| `generatedAt` | string | API gateway generation timestamp. |
| `supportedIds` | string[] | Optional local ids available to the current adapter result set. |

Current status taxonomy:

- `ok`: request succeeded.
- `not_found`: requested local resource id is not in the current adapter result set.
- `unsupported`: request shape is recognized but not supported by the current Phase 1 slice.
- `fail_closed`: data exists but required evidence is missing, ambiguous, or not yet confirmed.

Current error code taxonomy:

- `market_not_found`: implemented for detail lookup misses.
- `unsupported_market`: reserved until a route actually needs it.
- `out_of_scope`: reserved until a route actually needs it.

Typed error example:

```json
{
  "contractVersion": "ept-api-v1",
  "status": "not_found",
  "error": "market_not_found",
  "message": "Market not found in current Polymarket public-read adapter result set.",
  "generatedAt": "2026-04-23T00:00:00.000Z",
  "supportedIds": ["polymarket:mkt-btc-1h-demo"]
}
```

## `GET /scanner/top`

Purpose: publish read-only scanner candidates from the local fixture-backed adapter.

OpenAPI-like operation summary:

```yaml
method: GET
path: /scanner/top
successStatus: 200
responseBody: ScannerTopResponse
responseKind: scanner_top
```

Top-level response fields:

- `candidates`: ordered local scanner candidates.
- `meta`: `ScannerMeta` with shared success metadata plus scanner-specific metadata.

Candidate fields:

- `market`: normalized binary `EventMarket`.
- `fairValue`: `FairValueSnapshot` placeholder output.
- `tradeCandidate`: `TradeCandidate` placeholder output.
- `isPlaceholder`: always `true` in the current slice.

Scanner-specific metadata fields:

- `pricing`: `"pricing-engine-v0-placeholder"` or `"local-placeholder-fallback"`.
- `rejectedCount`: count of adapter records rejected by fail-closed normalization.
- `rejectionSummary`: grouped rejection reasons with sample upstream/local ids.
- `uncertainty`: TODO-style uncertainty strings that must not be silently resolved.

Placeholder semantics:

- `fairValue.fairProbabilityByOutcome.*.probability` is `null`.
- `fairValue.confidence` is `null`.
- `tradeCandidate.edge` is `null`.
- `isPlaceholder` is `true`.
- These fields are not model output, not trading advice, and not a real edge calculation.

## `GET /markets/:id/detail`

Purpose: publish a contract-backed Market Detail payload for read-only research inspection.

OpenAPI-like operation summary:

```yaml
method: GET
path: /markets/{id}/detail
pathParameters:
  id: URL-encoded normalized market id, for example polymarket:mkt-btc-1h-demo
successStatus: 200
errorStatuses:
  - 404 market_not_found
responseBody: MarketDetailResponse
responseKind: market_detail
```

Top-level response fields:

- `market`: normalized binary `EventMarket`.
- `relatedMarkets`: local related fixture market summaries.
- `researchReadiness`: outcome contract state, pricing placeholder state, classification source,
  open evidence gap count, and notes.
- `tokenTrace`: outcome token ids plus condition/question ids.
- `sourceTrace`: source id evidence items.
- `evidenceTrail`: classification/provenance evidence items.
- `openGaps`: uncertainty evidence items that must remain unresolved or fail closed.
- `meta`: shared success metadata.
- `candidate`: optional placeholder scanner candidate for this market.
- `book`: optional fixture-backed `OrderBookSnapshot`.

Path parameter:

- `id`: normalized local market id. URL-encode the value when it contains `:`.
- Example URL: `/markets/polymarket%3Amkt-btc-1h-demo/detail`.

## Fail-Closed And Placeholder Rules

- Missing or ambiguous evidence must stay visible as `TODO` strings or fail closed.
- Up/Down labels are preserved labels, not payoff specifications.
- Outcome labels must not be converted into payoff direction, trade side, or strategy side without
  confirmed evidence.
- No non-placeholder pricing may appear until payoff specification, reference/start/strike level,
  settlement source, comparator, tie rule, data freshness, and validation gates are documented and
  satisfied.
- The API contract exposes evidence gaps instead of silently filling them.
- `live_public` is a source mode, not approval to open live BTC/ETH 10m/1h classification or real
  pricing.

## Stable Contract Fields Versus Example Values

Stable local contract fields include:

- response shape and field names documented above;
- `contractVersion: "ept-api-v1"`;
- `responseKind` values;
- typed error `status` and `error` taxonomy;
- placeholder booleans and `null` pricing fields in the current slice;
- read-only flags and metadata.

Fixture/example values include:

- fixture market ids such as `polymarket:mkt-btc-1h-demo`;
- fixture token ids, condition ids, question ids, timestamps, liquidity, volume, bid/ask, spread,
  and book levels;
- snapshot `generatedAt` values fixed for tests;
- fixture rejection sample ids.

## Out Of Scope For RC-6

- Real-money trading.
- Private/authenticated Polymarket access.
- Predict.fun adapter.
- Binance adapter.
- Replay engine.
- Paper broker.
- Real pricing model.
- External network calls.
- Live fixture capture.
- API behavior changes.
- Snapshot changes.
- Page changes.
